# LMS 개발 릴리즈 계획

## 개요

본 문서는 `prd.md`와 `architecture_guide.md`를 기반으로 한 LMS 개발 릴리즈 계획을 정의합니다. 각 릴리즈는 동작 가능한 소프트웨어(Working Software)를 목표로 하며, 백엔드(Spring Boot)와 프론트엔드(React)의 통합을 포함합니다.

## 기술 스택 및 아키텍처

*   **Backend:** Spring Boot, Java, JPA, PostgreSQL
*   **Frontend:** React, TypeScript, Axios
*   **Architecture:** Clean Architecture, TDD, MSA (고려)
*   **CI/CD:** GitHub Actions

## 릴리즈 계획

### Release 1: 기본 설정 및 사용자 인증 (2주)

**목표:** 사용자 회원가입, 로그인, 권한 관리 등 핵심 인증 기능을 구현하고, 프로젝트의 기본 구조를 설정합니다.

| 구분 | 기능 ID | 기능 명칭 | 개발 내용 | 인수 기준 |
| --- | --- | --- | --- | --- |
| **Backend** | SFR-001, SFR-008, SFR-009 | 시스템 표준 준수, 회원가입, 협약사 가입 | - Spring Security 및 JWT를 이용한 인증/인가 구현<br>- User, Authority 엔티티 및 Repository 설계<br>- 회원가입(재직자/구직자), 협약사 가입 API 개발<br>- TDD 기반 테스트 코드 작성 | - API 요청 시 JWT 토큰 기반 인증 성공<br>- 역할(재직자, 구직자, 관리자)에 따른 API 접근 제어<br>- 회원가입 및 협약사 가입 기능 정상 동작 |
| **Frontend** | SFR-008, SFR-009 | 회원가입, 협약사 가입 | - React 프로젝트 초기 설정 (CRA, TypeScript)<br>- 로그인, 회원가입, 협약사 가입 페이지 UI 구현<br>- Axios를 이용한 백엔드 API 연동<br>- 상태 관리를 위한 Context API 또는 Recoil/Jotai 설정 | - 로그인/로그아웃 기능 정상 동작<br>- 회원가입 유형(재직자/구직자)에 따른 폼 변화<br>- API 연동을 통한 실제 회원가입 성공 |

- [ ] 로그인/로그아웃 기능이 정상적으로 동작하는가?
- [ ] 로그인 시 발급된 JWT 토큰으로 인증이 필요한 API에 접근할 수 있는가?

- [ ] 협약사 가입 신청이 가능한가?
- [ ] GitHub PR 생성 시 CI 빌드 및 테스트가 자동으로 실행되는가?
